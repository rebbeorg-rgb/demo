Resolving Nextcloud CORS Restrictions

By default, Nextcloud blocks incoming connection requests originating from external websites. To allow CloudStream to interact with Nextcloud directly from your browser, you have two quick options:

1 Option A: Configure Apache/Nginx (Recommended)

Add standard CORS headers directly in Nextcloud's main folder. Locate and append this block at the very end of your .htaccess file: 

<IfModule mod_headers.c>
    Header set Access-Control-Allow-Origin "*"
    Header set Access-Control-Allow-Methods "GET, POST, OPTIONS, PUT, DELETE, PROPFIND"
    Header set Access-Control-Allow-Headers "Authorization, Content-Type, Depth, OCS-APIRequest"
</IfModule>

2 Option B: Nextcloud Config.php Configuration

Navigate to your Nextcloud deployment directory and edit your config file (/config/config.php). Add the domain inside the array: 

'cors.allowed.domains' => [
  'https://your-streaming-website.com',
  '*' // Or use asterisk for wildcards
],

Note for Local Development/Testing:

If you are developing locally or just want to quickly try the interface immediately without altering your server configuration, you can use a browser extension like "Allow CORS: Access-Control-Allow-Origin" to safely whitelist request preflights temporarily.
